# Documentation

CrossExtend-KG mainline.

## Reading Order

1. `SYSTEM_DESIGN.md`
   Active architecture rules and runtime boundaries.
2. `PIPELINE_DATA_FLOW.md`
   End-to-end data flow from O&M markdown to exported graph artifacts.
3. `WORKFLOW_KG_DESIGN.md`
   Dual-layer graph design and evaluation boundary.
4. `DOWNSTREAM_EVALUATION.md`
   Workflow-centric downstream task design for paper-facing experiments.
5. `GRAPH_QUALITY_DIAGNOSIS.md`
   Current graph readability and structural quality analysis.
6. `OPEN_SOURCE_UPDATE_CN.md`
   Chinese summary of the current repository cleanup and open-source scope.

## Removed Historical Material

Old round-by-round optimization logs, one-off regression notes, and temporary
experiment reports are no longer part of the live repository contract. The
current repo surface is intentionally reduced to the active pipeline, active
evaluation, and active downstream design.
