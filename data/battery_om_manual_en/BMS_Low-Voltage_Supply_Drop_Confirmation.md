| T1 | For Velorian BMSCore-501 EV, record the low-voltage drop event as wake failure, controller reset, or intermittent communication loss, and note pack voltage plus the exact wake or ready transition that produced it. Save the loaded and unloaded values before the branch is opened. |
| T2 | Expose the BMS low-voltage feed path from the supply fuse through the wake relay or wake line to the BMS feed pin and ground eyelet, keeping the connectors seated until voltage checks are completed under load. The source, feed pin, and ground return have to be read as one electrical chain. |
| T3 | Measure voltage at the fuse source, relay output, BMS feed pin, and ground eyelet during the same wake transition, because a branch that looks healthy at rest can collapse only when current demand rises. Record the loaded drop at each point so the weak segment becomes explicit. |
| T4 | Inspect fuse clips, ring terminals, connector drag, relay contacts, and the ground eyelet for heat tint, oxide, or loose clamp load that would create a voltage loss without fully opening the circuit. Compare the worst reading with a neighboring low-voltage branch if a reference exists. |
| T5 | Hold the branch in the failed state and gently move the suspect fuse, connector, or ground path only if the loaded drop is movement-sensitive, then note exactly which item changes the voltage. The boundary belongs to the component that changes the reading, not to the whole feed path. |
| T6 | Correct only the failed fuse clip, BMS feed terminal, wake relay contact, or ground eyelet, then repeat the same wake transition with the branch restored to its final routed condition. The repair is not accepted if the loaded drop returns when the loom is clipped back into place. |
| T7 | Sign off the feed-path review only when the record names the exact failing component, the loaded voltage readings before and after repair, and the final seating condition at both feed and ground ends. The release boundary is a BMS supply path that holds the required voltage during wake demand with no reset, no communication loss, and no movement-sensitive drop. |
