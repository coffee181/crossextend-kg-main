| T1 | On Aurex BMSCore-410, record the removed controller part number, serial number, coding status, pack identifier, and fault memory before the housing is unbolted, because commissioning the replacement depends on the exact as-removed identity. | T2 | Make the pack safe and expose the BMS housing, low-voltage connectors C1 and C2, voltage-sense headers J3 and J4, CAN branch, ground eyelet, and vent membrane while labeling every interface before disconnecting it. Keep the harness in its natural routed position so a preload issue is not introduced during removal. |
| T3 | Compare the replacement controller with the removed unit for connector keying, mounting-boss height, vent orientation, software family label, and sensor-header population before it is mounted. A controller that fits the bracket but carries the wrong sense-header layout or firmware family creates a different fault class than a failed original unit. |
| T4 | Install the replacement on clean mounting lands, reconnect the sense headers, CAN branch, supply connector, and ground eyelet in sequence, and confirm that no harness presses against the housing corner or cover edge. Record the initial supply voltage and ground continuity at the controller after installation. |
| T5 | Program coding and variant data as required, then verify wake-up, CAN presence, cell-voltage visibility, pack-current reading, and temperature-channel plausibility before the enclosure is closed. Coding is not complete until the live data proves the controller is communicating on the correct pack configuration. |
| T6 | Compare one voltage-sense channel, one temperature channel, and the CAN state with known-good neighboring references so the first proof is anchored to real pack behavior rather than to the absence of a DTC alone. Use the same wake or ready state for every comparison in this step. |
| T7 | Refit covers and supports and repeat the wake and live-data check because connector preload, carrier fit, or ground-eyelet tension can change once the housing is fully enclosed. The installed controller has to survive its final harness path from connector to first clip. |
| T8 | Close the controller replacement only when the note lists the removed and installed controller identifiers, coding result, verified live-data channels, and final physical condition of the connector, ground, and housing interfaces. The release boundary is a coded controller that wakes correctly, reports coherent pack data, and remains mechanically neutral after the enclosure is restored. |
