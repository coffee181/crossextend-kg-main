| T1 | Archive the pre-replacement BMS baseline on Novaris Logic-522 AWD by saving the original controller serial number, software level, cell map, learned balancing state, wake configuration, contactor-permission status, and full DTC snapshot before replacement begins. This intake record has to preserve both the electrical identity and the as-found control state so the new controller can later be judged against one stable baseline instead of against memory alone. |
| T2 | Depower the battery electronics, apply ESD protection, and open the BMS enclosure so the main controller, sense connectors, communication connector, low-voltage feed, ground braid, standoff hardware, thermal pad area, and lid seal path are visible together before anything is unplugged. Keep the enclosure interior undisturbed long enough to read harness dress, connector clocking, and the board-to-baseplate relationship in one view before the original package geometry is lost. |
| T3 | Label every sense harness, low-voltage plug, and shield bond by position, then inspect the controller header pins, connector locks, ground eyelet, enclosure corners, and nearby standoffs for bent hardware, moisture residue, pin recession, or previous service marks that could complicate the replacement result. Any damaged lock tab, oxidized eyelet face, or recessed header terminal has to be called out now so a later communication or channel fault is not blamed on the new controller incorrectly. |
| T4 | Remove the controller fasteners in sequence, lift the old controller clear without dragging the header pins, and inspect the thermal interface pad, standoff seats, mounting bosses, and ground contact points on the baseplate for debris, distortion, corrosion film, or uneven seating witness that would keep the new board from sitting flat. Compare the witness pattern from corner to corner so the mounting boundary can be assigned to one warped boss, one contaminated standoff seat, or one disturbed ground pad instead of to general poor fit. |
| T5 | Install the replacement controller, confirm full seating on every standoff, tighten the fasteners to specification, and verify that no board corner is floating, no harness is trapped under the edge, the ground braid lands squarely, and the thermal interface pad remains correctly aligned across the heat-transfer area. The board should sit with a uniform gap and stable contact at every boss so heat transfer, ground continuity, and connector alignment are all restored together rather than corrected one at a time. |
| T6 | Reconnect each sense connector, communication connector, and ground braid in the documented order, checking secondary locks, terminal seating, wire dress, and connector orientation so the enclosure can close without side-loading the plugs, twisting the harnesses, or pulling on the board header during final lid installation. Read the harness route from connector body to clip point and from clip point to enclosure wall so no branch is left crossing a standoff or pressing on the lid seal path. |
| T7 | Reapply low-voltage power, program or pair the replacement controller as required, and confirm that every cell-voltage group, temperature channel, wake state, balancing permissive, contactor request, and communication parameter populates normally before the enclosure lid is returned. Compare the post-programming channel map with the saved baseline so a missing module, shifted thermistor group, or altered balancing state is separated from the physical replacement itself. |
| T8 | Fit the enclosure lid, verify the seal path is clean and continuous, and run a wake-sleep-wake cycle while observing controller online status, module detection, balancing status, low-voltage draw, and fault memory for any mismatch created by the replacement, the programming step, or the reconnection order. The release window is not just one successful wake event but a stable controller that remains online through closure, sleep entry, and the next commanded wake without dropping a channel or reflagging a learned-state conflict. |
| T9 | Release the job only after the record ties the new controller serial number, software configuration, connector verification, enclosure closure, baseline comparison, and final functional result to the same vehicle work order with no unresolved channel mismatch, wake error, or communication inconsistency. The accepted boundary is a replacement BMS that sits flat on its thermal and ground interfaces, reports the full module map correctly, and holds normal wake behavior after the enclosure is fully resealed. |
