| T1 | On Helion Rotary-400, record the rotary-table complaint as no brake release, delayed release, or drift after clamp, and note table angle, fixture load, and whether the issue appears at one sector or across the full rotary range. Capture the exact clamp and release condition before the actuator is opened. |
| T2 | Expose brake actuator BRK-A1 or hydraulic or pneumatic line, pressure or voltage feed, mechanical brake housing, release switch LS-BRK if fitted, and local hose or cable support while preserving the as-found routing. Keep the actuation path and the mechanical brake pack visible together so command and actual release can be compared directly. |
| T3 | Command brake release and compare electrical or pressure command with actual table response and release-travel measurement at actuator BRK-A1. If the signal reaches the actuator but the table still drags or the travel remains short, keep the boundary on the brake pack, piston, or release circuit rather than on the controller output alone. |
| T4 | Inspect connector or fitting retention, actuator travel, release spring or return path, and brake housing witness from upper feed point to lower brake pack. A sticky brake pack, leaking release line, or weak command branch can all create delayed release, but they show different evidence in stroke, pressure, and table movement. |
| T5 | Repeat release and clamp cycles at several rotary positions and compare actuator stroke and table behavior with the pre-repair condition. This keeps the diagnosis on one real brake boundary instead of treating a single favorable angle as full proof of correction. |
| T6 | Correct only the failed coil, line, piston, or brake-pack issue, then restore hoses, cables, and guards to their original production path. Confirm that the branch does not pull the actuator off center and that the brake pack returns fully to clamp when the command is removed. |
| T7 | Run the same release and clamp sequence again after closure and inspect the corrected boundary for renewed drift, short stroke, or leakage because rotary brake faults often return once the table settles under load. The accepted result has to survive the final machine configuration, not only an open-service check. |
| T8 | Record the exact failed boundary, measured release response at BRK-A1, tested table positions, and final table behavior in one traceable note. The report should show whether the problem was electrical, pneumatic or hydraulic, or mechanical inside the brake pack. |
| T9 | Close the work only after the report names the exact hardware or control boundary and ties it to repeated release and clamp behavior under the original load condition. Accept the repair when the rotary brake releases and holds repeatably with no renewed delay or drift. |
