## Core Constraints
1. The backbone is frozen. Never invent a new backbone concept.
2. You may only choose from these routes: __ALLOWED_ROUTES__.
3. `allow_free_form_growth=__ALLOW_FREE_FORM_GROWTH__`.
4. Return `reuse_backbone` only when the candidate label exactly matches a backbone concept.
5. Otherwise, if the candidate should enter the graph, use `vertical_specialize` with a valid backbone `parent_anchor`.
6. The output must be valid JSON and strictly match the return schema.
7. Reject person names and document titles. They belong in provenance, not as graph nodes.
8. `routing_features.semantic_type_hint` is a soft prior, not a hard label. Use it to improve judgment, but override it when the candidate semantics clearly disagree.
9. `routing_features.is_task_candidate=true` means the candidate is a true step-task node and should normally attach under `Task`.
10. `routing_features.task_step_id` only means the candidate is referenced in a step context if `is_task_candidate` is false; it does not make the candidate a `Task`.
11. If you reject a candidate, you must set `route=reject`, `accept=false`, `admit_as_node=false`, and use exactly one reject reason from this closed set:

## Attachment Rules
12a. A label that is itself a true O&M step candidate should normally attach under `Task`.
12b. A component, part, connector, hose, clip, manifold, seal, sensor, or hardware item should normally attach under `Component`.
12c. A named machine, vehicle, pack, cabinet, line, or platform should normally attach under `Asset`.
12d. An observed result, warning, residue path, wetting, level, offset, reading, or measurement outcome should normally attach under `Signal`.
12e. A persistent condition such as dry, wet after shutdown, fully latched, under circulation, or naturally supported should normally attach under `State`.
12f. A defect, crack, leak, distortion, incomplete insertion, side-load root cause, or failure mode should normally attach under `Fault`.
12g. If `relation_families` contains only `propagation` and/or `communication`, do not attach the candidate under `Task` unless the candidate is a true step-task node.
12h. Being mentioned in a step or having `task_step_id` does not by itself justify `Task`.

## Quality Rules
13. Observation-like labels may be admitted only when they are grounded enough to function as reusable `Signal` or `State` nodes.
14. Reject low-value fragments, bookkeeping residue, or overly tiny modifiers as `low_graph_value`.
15. Components and other industrial entities should be admitted only when they both anchor to the backbone and participate in the relation chain or step evidence with graph value.
16. Historical context may help preserve temporal consistency, but never accept a candidate solely because it appeared before.
